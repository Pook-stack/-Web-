## 3. 数据库设计

### 3.1 clubs 表
```sql
CREATE TABLE clubs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name TEXT NOT NULL,
    game TEXT NOT NULL,
    description TEXT,
    detailed_description TEXT,
    contact TEXT,
    member_requirement TEXT,
    price_range TEXT,
    special_services TEXT[],
    icon_url TEXT,
    cover_image_url TEXT,
    founded_year TEXT,
    member_count INTEGER DEFAULT 0,
    playmate_count INTEGER DEFAULT 0,
    rating NUMERIC(3,1) DEFAULT 0,
    review_count INTEGER DEFAULT 0,
    mission TEXT,
    activity_frequency TEXT,
    status TEXT DEFAULT 'pending',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 3.2 users 表
```sql
CREATE TABLE users (
    id TEXT PRIMARY KEY,
    username TEXT NOT NULL,
    role TEXT DEFAULT 'member',
    status TEXT DEFAULT 'active',
    avatar_url TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    banned_at TIMESTAMP WITH TIME ZONE
);
```

### 3.3 club_members 表
```sql
CREATE TABLE club_members (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    club_id BIGINT NOT NULL REFERENCES clubs(id) ON DELETE CASCADE,
    user_id TEXT NOT NULL,
    role TEXT DEFAULT 'member',
    joined_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    UNIQUE(club_id, user_id)
);
```

### 3.4 applications 表
```sql
CREATE TABLE applications (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    club_id BIGINT NOT NULL REFERENCES clubs(id) ON DELETE CASCADE,
    user_id TEXT NOT NULL,
    status TEXT DEFAULT 'pending',
    applied_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    processed_at TIMESTAMP WITH TIME ZONE
);
```

### 3.5 notifications 表
```sql
CREATE TABLE notifications (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    type VARCHAR(50) NOT NULL CHECK (type IN ('application_approved', 'application_rejected', 'club_joined', 'system_notice', 'club_created')),
    content TEXT NOT NULL,
    is_read BOOLEAN NOT NULL DEFAULT false,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    metadata JSONB
);
```

### 3.6 索引
```sql
CREATE INDEX idx_clubs_status ON clubs(status);
CREATE INDEX idx_clubs_game ON clubs(game);
CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_club_members_club_id ON club_members(club_id);
CREATE INDEX idx_applications_club_id ON applications(club_id);
CREATE INDEX idx_notifications_user_id ON notifications(user_id);
```

## 8. 常见问题与解决方案

### 8.1 数据库连接失败
**问题**: `net::ERR_NAME_NOT_RESOLVED`
**解决方案**: 
1. 检查网络连接
2. 确认Supabase URL配置正确
3. 等待DNS传播（5-30分钟）
4. 尝试刷新DNS缓存

### 8.2 通知重复显示
**问题**: 每次打开网站都显示相同的通知
**解决方案**:
1. 使用localStorage保存已读状态
2. 移除所有模拟数据
3. 从数据库获取真实通知

### 8.3 页面卡顿
**问题**: 点击申请加入后页面无响应
**解决方案**:
1. 使用异步操作替代阻塞式window.confirm
2. 实现自定义确认对话框
3. 异步处理localStorage操作

### 8.4 DNS解析失败
**问题**: 无法访问Supabase域名
**解决方案**:
1. 检查DNS配置
2. 确认A记录和CNAME记录正确
3. 等待DNS传播
4. 尝试使用手机热点

### 8.5 Vercel部署失败
**问题**: 部署时出现错误
**解决方案**:
1. 检查环境变量配置
2. 查看Build Log定位错误
3. 确保代码无语法错误
4. 重新部署
