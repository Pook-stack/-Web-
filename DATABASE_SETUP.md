## 表结构

### 1. `clubs` 表 - 俱乐部信息

```sql
CREATE TABLE clubs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name TEXT NOT NULL,
    game TEXT NOT NULL,
    description TEXT,
    detailed_description TEXT,
    contact TEXT,
    member_requirement TEXT,
    price_range TEXT,
    special_services TEXT[],
    icon_url TEXT,
    cover_image_url TEXT,
    founded_year TEXT,
    member_count INTEGER DEFAULT 0,
    playmate_count INTEGER DEFAULT 0,
    rating NUMERIC(3,1) DEFAULT 0,
    review_count INTEGER DEFAULT 0,
    mission TEXT,
    activity_frequency TEXT,
    status TEXT DEFAULT 'pending',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_clubs_status ON clubs(status);
CREATE INDEX idx_clubs_game ON clubs(game);
CREATE INDEX idx_clubs_created_at ON clubs(created_at);
```

### 2. `users` 表 - 用户信息

```sql
CREATE TABLE users (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username TEXT NOT NULL,
    role TEXT DEFAULT 'member',
    status TEXT DEFAULT 'active',
    avatar_url TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    banned_at TIMESTAMP WITH TIME ZONE
);

-- 索引
CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_users_role ON users(role);
```

### 3. `club_members` 表 - 俱乐部成员关系

```sql
CREATE TABLE club_members (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    club_id BIGINT NOT NULL REFERENCES clubs(id) ON DELETE CASCADE,
    user_id BIGINT NOT NULL,
    role TEXT DEFAULT 'member',
    joined_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    UNIQUE(club_id, user_id)
);

-- 索引
CREATE INDEX idx_club_members_club_id ON club_members(club_id);
CREATE INDEX idx_club_members_user_id ON club_members(user_id);
```

### 4. `applications` 表 - 加入申请

```sql
CREATE TABLE applications (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    club_id BIGINT NOT NULL REFERENCES clubs(id) ON DELETE CASCADE,
    user_id BIGINT NOT NULL,
    status TEXT DEFAULT 'pending',
    applied_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    processed_at TIMESTAMP WITH TIME ZONE
);

-- 索引
CREATE INDEX idx_applications_club_id ON applications(club_id);
CREATE INDEX idx_applications_user_id ON applications(user_id);
CREATE INDEX idx_applications_status ON applications(status);
```

## RLS（行级安全）策略

```sql
-- 启用 RLS
ALTER TABLE clubs ENABLE ROW LEVEL SECURITY;
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE club_members ENABLE ROW LEVEL SECURITY;
ALTER TABLE applications ENABLE ROW LEVEL SECURITY;

-- 策略：所有用户可以查看已批准的俱乐部
CREATE POLICY "Public can view approved clubs" 
    ON clubs FOR SELECT 
    USING (status = 'approved');

-- 策略：所有用户可以创建俱乐部申请
CREATE POLICY "Users can create clubs" 
    ON clubs FOR INSERT 
    WITH CHECK (true);

-- 策略：所有用户可以申请加入俱乐部
CREATE POLICY "Users can apply to clubs" 
    ON applications FOR INSERT 
    WITH CHECK (true);
```

## Supabase 设置

### 1. 创建项目

1. 访问 https://supabase.com
2. 创建新项目 `ruamzahurunequan`
3. 设置数据库密码（记住它）

### 2. 获取 API 密钥

1. 进入项目设置 → API
2. 复制 `Project URL` （应该是：`https://ruamzahurunequan.supabase.co`）
3. 复制 `anon public key`

### 3. 运行 SQL

1. 进入 SQL Editor
2. 复制上面的 SQL 代码
3. 点击 Run
4. 确认表已创建

### 4. 验证

在 Table Editor 中检查：
- [ ] clubs 表存在
- [ ] users 表存在
- [ ] club_members 表存在
- [ ] applications 表存在
- [ ] 有初始数据

## 测试连接

运行项目后检查浏览器控制台：
- ✅ "✅ 数据库连接成功" - 连接正常
- ⚠️ "数据库连接失败，使用离线模式" - 检查数据库设置

## 故障排查

### 问题：连接失败

1. 检查 Supabase 项目是否在运行
2. 验证 API 密钥是否正确
3. 检查是否有网络阻止
4. 确认表是否已创建

### 问题：表不存在

确保已在 SQL Editor 中运行了所有 SQL 代码

### 问题：权限错误

检查 RLS 策略是否正确设置
