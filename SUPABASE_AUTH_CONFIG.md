# Supabase 邮箱验证配置指南

## 配置步骤

### 1. 登录 Supabase 控制台
访问 https://supabase.com/dashboard 并登录您的账户。

### 2. 进入项目设置
- 选择您的项目
- 导航至 **Authentication** → **Settings**

### 3. 配置 Email Confirmation（必须开启）
- 在 **Email Confirmation** 部分
- 确保 **Confirm email** 选项处于 **启用状态**（开关为蓝色）
- **Email confirmation type** 选择 **Manual**（手动确认）

### 4. 配置 Redirect URLs
在 **Redirect URLs** 部分，添加以下 URL：

```
http://localhost:5178/**
http://localhost:5175/**
http://localhost:5179/**
```

> **注意**：如果您使用其他端口开发，也需要添加相应的 URL

### 5. 配置 Site URL（可选但建议）
- 在 **Site URL** 输入框中填写您的应用 URL
- 开发环境：`http://localhost:5178`

### 6. 配置 SMTP（可选，用于自定义邮件发送）
如果需要自定义邮件发送服务器，可以配置 SMTP 设置。

## 测试验证流程

### 步骤 1：启动开发服务器
```bash
npm run dev
```

### 步骤 2：注册新用户
1. 访问注册页面
2. 输入真实可用的邮箱地址（如 Gmail、Outlook）
3. 设置密码并提交注册

### 步骤 3：检查验证邮件
1. 登录您的邮箱
2. 查找来自 Supabase 的验证邮件
3. 点击验证链接

### 步骤 4：验证成功
点击链接后应跳转到验证成功页面，显示"邮箱验证成功！您的账号已激活，可以登录了。"

## 常见问题排查

### 问题 1：收不到验证邮件
**可能原因**：
- Supabase 的 Confirm email 选项未开启
- Redirect URLs 配置不正确
- 邮件被归类到垃圾邮件文件夹
- 使用了临时邮箱或一次性邮箱

**解决方案**：
1. 确认上述配置步骤已完成
2. 检查邮箱的垃圾邮件文件夹
3. 使用主流邮箱服务（Gmail、Outlook、QQ邮箱等）

### 问题 2：验证链接无效
**可能原因**：
- 链接已过期（默认24小时有效）
- 链接被篡改
- 用户已完成验证

**解决方案**：
1. 重新注册获取新链接
2. 检查链接是否完整复制

### 问题 3：验证后无法登录
**可能原因**：
- 密码设置不符合要求
- 用户状态异常

**解决方案**：
1. 使用"忘记密码"功能重置密码
2. 检查 Supabase 用户列表确认用户状态

## 安全注意事项

1. **生产环境配置**：部署到生产环境时，务必更新 Redirect URLs 为正式域名
2. **HTTPS**：生产环境必须使用 HTTPS
3. **环境变量**：敏感配置（如 API Key）应存储在环境变量中，不要硬编码
4. **Rate Limiting**：Supabase 有请求频率限制，避免短时间内大量注册

## 配置验证

配置完成后，可以通过以下方式验证：

1. 检查 Supabase 控制台的 Authentication Settings 页面
2. 查看是否有任何配置警告
3. 测试完整的注册→验证→登录流程
